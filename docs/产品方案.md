# 「BI 仪表盘分析插件」方案确认文档

## 1. 方案概述

### 1.1 业务场景

当前客户希望在 ONES 内构建一套更灵活的 BI 仪表盘分析能力，用于围绕项目、迭代、工作项、缺陷、工时、成员等研发数据进行自助分析、可视化看板搭建、明细下钻和报表共享。

标准效能管理或固定报表通常存在指标数量、分析维度、图表形态、卡片布局和自定义字段分析能力受限的问题。通过插件方式补充独立 BI 仪表盘能力，可以让项目经理、PMO、研发管理者和团队负责人按自身管理口径配置仪表盘，并在同一页面内完成指标查看、趋势分析、异常定位和明细追踪。

### 1.2 用户故事

作为项目管理者，我希望在 ONES 中基于项目、迭代、工作项、缺陷、工时和成员等数据，自定义仪表盘、图表、筛选条件和下钻明细，以便持续跟踪项目进展、交付效率和质量风险。

### 1.3 本次范围

本次方案覆盖 BI 仪表盘分析插件的完整规划范围，并按优先级拆分为 P0、P1、P2：

- P0：数据源与数据范围、数据集与语义层、指标管理、维度与字段分析、仪表盘管理、仪表盘权限、卡片与布局、基础图表、筛选排序、下钻明细、数据刷新与性能。
- P1：发布与版本、仪表盘模板、全局筛选器、图表联动、分享订阅与导出、审计日志、资产使用统计。
- P2：告警监控、高级图表、AI 洞察、复杂数据血缘、智能推荐能力。

本方案正文仅覆盖本次范围内内容，未纳入本次范围的事项不在正文展开说明。

### 1.4 前置依赖

- 客户 ONES 环境中已启用 ONES Project，并存在项目、迭代、工作项、状态、字段、成员等基础数据。
- 工时分析依赖客户环境中已启用工时能力，且工时数据可通过 OpenAPI 或可用查询能力获取。
- 成员、部门、用户组分析依赖客户环境中存在对应组织数据，且当前账号具备查看权限。
- 工作项历史状态、状态停留时长、流转耗时等分析依赖工作项变更记录或状态流转记录可获取。
- 报表通知、订阅、告警推送依赖客户环境已配置可用通知渠道。
- 导出 PDF、Excel、图片等格式需要结合客户环境、浏览器能力和插件运行限制确认最终支持范围。

## 2. 详细功能设计

### 2.1 插件管理

#### 2.1.1 团队插件接入

- 入口：`【配置中心 - 应用管理 - 团队插件管理】`
- 名称：`BI 仪表盘分析插件`
- 描述：`用于在 ONES 中基于项目、迭代、工作项、缺陷、工时和成员等数据构建自定义 BI 仪表盘、图表分析、下钻明细和报表共享能力。`
- 权限：`插件管理员`

#### 2.1.2 插件启用、停用、卸载管理

- 启用：插件管理员在团队插件管理中启用插件后，系统增加 BI 仪表盘入口和插件配置页。启用后仅具备插件访问权限的成员可进入插件页面。
- 停用：插件停用后，普通成员不可继续访问 BI 仪表盘入口；已保存的仪表盘、数据集、卡片配置和权限配置保留。
- 卸载：插件卸载后，插件入口移除。是否保留插件数据按 ONES 插件卸载规则和客户实施策略执行，卸载前应提示管理员备份重要配置。

#### 2.1.3 插件配置页

- 插件启用后，增加插件配置页。
- 配置页用于维护数据获取方式、默认刷新策略、导出开关、通知渠道开关、缓存策略、管理员权限和异常日志查看入口。
- 插件配置仅允许插件管理员访问。

#### 2.1.4 插件日志

- 插件提供独立日志能力。
- 日志记录范围包括：仪表盘创建、编辑、删除、复制、转让、分享、导出、刷新失败、查询失败、告警触发、订阅发送结果和插件配置变更。
- 插件管理员可按操作类型、操作者、时间范围、对象名称筛选日志。

### 2.2 数据源与数据范围

- 功能目标：为仪表盘分析提供统一的数据选择入口，明确每类图表可使用的数据对象和字段范围。
- 交互说明：
  - 用户新建数据集或卡片时，先选择数据来源。
  - 系统展示可选数据对象，包括项目、迭代、工作项、工作项类型、状态、字段、成员、工时、工作项关联关系。
  - 用户选择数据对象后，系统展示可用字段、字段类型、是否可作为维度、是否可作为指标、是否可作为筛选条件。
  - 用户保存配置后，该数据范围用于后续指标、维度、筛选和明细表配置。
- 异常与边界处理：
  - 当数据源不可用时，页面展示失败原因和重试入口。
  - 当字段在客户环境中不存在时，展示字段缺失提示，不阻塞其他可用字段分析。
  - 当用户无权访问某项目或工作项时，该数据不进入统计结果。

### 2.3 数据集与语义层

- 功能目标：将底层 ONES 数据封装为可复用的数据集，避免每张卡片重复配置字段、过滤条件和口径。
- 交互说明：
  - 用户进入数据集管理页面，可新建、编辑、删除数据集。
  - 新建数据集时，用户选择数据对象、基础筛选条件、可用字段和字段别名。
  - 用户可为字段配置展示名称、字段类型、默认聚合方式、是否启用、口径说明。
  - 数据集保存后，可被多个仪表盘和卡片引用。
  - 数据集被修改时，系统提示受影响的卡片数量和仪表盘范围。
- 异常与边界处理：
  - 已被卡片引用的数据集不允许直接删除，需先解除引用或确认影响范围。
  - 字段类型变化后，依赖该字段的图表需展示配置异常。
  - 复杂计算字段仅支持在可获取数据范围内计算，不承诺跨系统补齐缺失数据。

### 2.4 指标管理

- 功能目标：提供可复用、可解释的指标配置能力，支持多指标分析和统一口径管理。
- 交互说明：
  - 用户在数据集内或卡片配置中添加指标。
  - 指标支持计数、去重计数、求和、平均、最大、最小、占比等聚合方式。
  - 用户可配置指标名称、单位、精度、小数位、百分比展示和口径说明。
  - 图表配置支持多指标，不沿用现有效能管理 5 个指标限制。
  - 用户可在趋势类图表中配置当期值、累计值、同比、环比等展示方式。
- 异常与边界处理：
  - 当指标字段不支持所选聚合方式时，系统阻止保存并提示原因。
  - 当多指标时间口径不一致时，系统提示推荐使用自然时间作为横轴。
  - 指标口径仅基于可获取数据计算，不对不可获取历史数据进行推断。

### 2.5 维度与字段分析

- 功能目标：支持基于系统字段和自定义字段进行多维分析，覆盖研发管理中常用的状态、负责人、优先级、项目、迭代、类型、日期等维度。
- 交互说明：
  - 用户在卡片配置中选择一个或多个维度。
  - 系统支持单选、多选、日期、人员、数字、文本等常见字段作为维度或筛选条件。
  - 多选字段支持拆分聚合，例如 A/B/C 三个选项分别计数。
  - 用户可调整维度显示顺序、多维排序规则、Top N / Bottom N 展示规则。
- 异常与边界处理：
  - 文本字段作为维度时，如枚举值过多，系统提示建议使用 Top N 或筛选条件收敛结果。
  - 多选字段拆分聚合时，同一工作项命中多个选项会分别计入各选项，需在口径说明中展示。
  - 字段被删除或权限不可见时，卡片进入配置异常状态。

### 2.6 仪表盘管理

- 功能目标：支持用户创建、管理、分类、复制和转让 BI 仪表盘。
- 交互说明：
  - 用户进入 BI 仪表盘列表，列表包含全部、我创建的、共享给我的、最近查看、收藏等视图。
  - 用户可新建仪表盘，填写名称、描述、分类、可见范围。
  - 用户可编辑、删除、复制、归档、恢复仪表盘。
  - 仪表盘支持分类和层级管理。
  - 仪表盘支持 tab 切换，用于在同一仪表盘内组织不同分析主题。
  - 仪表盘展示 owner，owner 可将仪表盘转让给其他成员，转让后新 owner 可编辑。
  - 支持单个仪表盘容纳更多卡片，目标按 100 张卡片验证。
- 异常与边界处理：
  - 删除仪表盘需二次确认。
  - 已归档仪表盘不在默认列表展示，但可在归档列表恢复。
  - 当仪表盘卡片数量较多时，系统分批加载卡片数据，避免页面一次性请求过多。

### 2.7 仪表盘权限

- 功能目标：在插件内提供最小可用权限模型，并确保数据结果不越过 ONES 原有数据权限。
- 交互说明：
  - 仪表盘权限角色包括 owner、编辑者、查看者。
  - owner 可管理仪表盘、卡片、权限、转让、删除。
  - 编辑者可编辑仪表盘内容和卡片配置，但不可转让 owner。
  - 查看者可查看仪表盘和下钻明细，不可编辑配置。
  - owner 可按成员、部门、用户组共享仪表盘。
  - 系统支持查看、编辑、复制、删除、导出权限控制。
- 异常与边界处理：
  - 插件权限只控制仪表盘资产，不扩大用户在 ONES 中的数据访问范围。
  - 用户查看仪表盘时，只能看到其在 ONES 中有权限访问的数据。
  - 无权限访问仪表盘时，展示无权限提示。
  - 导出权限可单独关闭，关闭后用户不可导出明细或仪表盘。

### 2.8 发布与版本

- 功能目标：降低编辑仪表盘对查看者的影响，支持草稿、发布和历史回滚。
- 交互说明：
  - owner 或编辑者编辑仪表盘时，系统保存为草稿。
  - 草稿未发布前，查看者仍看到已发布版本。
  - 编辑者完成调整后点击发布，填写发布备注，发布后查看者看到新版本。
  - 系统保存版本历史，owner 可查看历史版本并回滚。
  - 复制仪表盘时，用户可选择是否复制权限、筛选器、数据集配置。
- 异常与边界处理：
  - 当草稿中存在异常卡片时，不允许发布。
  - 回滚历史版本后，若引用字段已不存在，相关卡片进入配置异常状态。

### 2.9 卡片与布局

- 功能目标：支持用户通过卡片构建仪表盘内容，并灵活调整卡片布局和样式。
- 交互说明：
  - 用户在仪表盘编辑态中新建卡片，选择卡片类型和数据集。
  - 支持新建、编辑、复制、删除卡片。
  - 支持拖拽调整卡片位置和尺寸。
  - 数字卡支持更小尺寸，避免固定 3x3。
  - 卡片标题字数限制放开，但超长标题在展示态自动换行或省略。
  - 卡片间支持分隔符或分区能力，用于按业务主题组织页面。
  - 图表拖拽条支持显隐，调整后保存。
  - 卡片支持基础样式配置，包括标题、内容颜色、字号、字体、背景和边框。
- 异常与边界处理：
  - 卡片配置不完整时，不允许保存。
  - 卡片数据查询失败时，卡片内展示失败状态和重试入口。
  - 卡片复制到其他仪表盘时，如目标仪表盘无对应数据权限，展示提示。

### 2.10 图表与可视化

- 功能目标：提供覆盖常见 BI 分析场景的图表类型和展示配置。
- 交互说明：
  - 支持文本卡、数字指标卡、柱状图、堆叠柱状图、百分比堆叠柱状图、折线图、柱状图 + 折线图组合图、饼图、环形图、面积图、表格、透视表。
  - 趋势图支持按自然日展示。
  - 趋势图支持按月累计趋势。
  - 支持隐藏 0 值。
  - 饼图支持按状态自定义顺序。
  - 用户配置图表时可实时预览。
  - 支持自定义配色、图例显隐、坐标轴、标签、单位和数值格式。
  - 燃尽图、燃起图作为研发效能场景增强图表。
  - 漏斗图、热力图、散点图作为后续增强图表。
- 异常与边界处理：
  - 当所选维度或指标不满足图表要求时，系统提示调整配置。
  - 图表无数据时展示空态，不展示错误态。
  - 图表数据过多时，提示使用筛选、Top N 或表格展示。

### 2.11 筛选、排序与查询

- 功能目标：支持用户在仪表盘和卡片层面快速收敛分析范围。
- 交互说明：
  - 支持时间范围筛选。
  - 支持筛选条件组。
  - 支持筛选项模糊匹配。
  - 支持所属项目与所属迭代级联筛选。
  - 支持按创建时间、完成时间、优先级、状态、负责人等字段排序。
  - 支持多个维度排序。
  - 支持仪表盘级全局筛选器。
  - 全局筛选器可配置作用于全部卡片或指定卡片。
  - 支持保存默认筛选条件、清空筛选、重置为默认筛选。
- 异常与边界处理：
  - 当全局筛选器字段不适用于某卡片数据集时，该卡片不受该筛选器影响，并在配置中展示说明。
  - 单指标可按完成时间作为时间横轴，多指标优先按自然时间展示。
  - 复杂筛选条件组需在保存前校验字段类型和运算符匹配。

### 2.12 图表联动与交互分析

- 功能目标：支持从汇总指标快速定位到影响结果的明细数据。
- 交互说明：
  - 支持数据下钻。
  - 用户点击柱状图 bar 可进入对应明细。
  - 用户点击折线图点位可进入对应明细。
  - 用户点击透视表单元格或数字卡可进入明细。
  - 支持图表间联动过滤。
  - 支持图例点击隐藏或显示。
  - 支持下钻路径配置，例如项目 -> 迭代 -> 负责人 -> 明细。
  - 支持上卷返回。
  - 用户可查看当前卡片统计口径、筛选条件和数据来源。
- 异常与边界处理：
  - 当点击区域无法映射到明细条件时，不触发下钻。
  - 下钻明细仍需按用户 ONES 数据权限过滤。
  - 图表联动可能导致其他卡片无数据，无数据时展示空态。

### 2.13 明细表

- 功能目标：承接图表下钻结果，展示可追踪、可跳转、可筛选的明细数据。
- 交互说明：
  - 明细表支持自定义表头。
  - 支持列宽拖拽调整、排序、筛选、字段显隐。
  - 支持固定表头和冻结列。
  - 明细中的工作项支持跳转到 ONES 工作项详情。
  - 支持明细数据导出，格式按可行性评估确认。
  - 支持大数据量分页加载。
- 异常与边界处理：
  - 用户无导出权限时，不展示导出入口。
  - 明细字段缺失时，该列展示为空并提示字段不可用。
  - 导出数据量超过限制时，提示缩小筛选范围或使用异步导出。

### 2.14 数据刷新与性能

- 功能目标：保障仪表盘在多卡片、大数据量场景下可用，并减少对 ONES 接口的压力。
- 交互说明：
  - 支持手动刷新。
  - 支持定时刷新。
  - 支持卡片级刷新状态和仪表盘级刷新。
  - 卡片展示最近更新时间。
  - 支持加载中、无数据、失败、权限不足等状态。
  - 支持 100 张卡片仪表盘的性能验证。
- 异常与边界处理：
  - 查询超时后展示超时提示和重试入口。
  - 后台查询失败时记录失败日志。
  - 高频刷新需受系统配置限制，避免影响 ONES 服务稳定性。

### 2.15 分享、订阅与导出

- 功能目标：支持仪表盘在团队内分发和沉淀。
- 交互说明：
  - owner 可按成员、部门、用户组共享仪表盘。
  - 支持仪表盘收藏和最近查看。
  - 支持定时报表订阅。
  - 支持邮件、飞书、企业微信、钉钉等通知渠道，按客户环境可用性确定。
  - 支持导出图片、PDF、Excel，具体格式按可行性确认。
  - 支持仪表盘快照。
  - 用户可管理和取消自己的订阅。
- 异常与边界处理：
  - 通知渠道未配置时，不允许创建对应渠道订阅。
  - 导出失败时记录失败原因并提示用户重试。
  - 快照数据仅代表生成时刻，不随源数据变化自动更新。

### 2.16 告警与监控

- 功能目标：对关键指标异常变化进行主动提醒。
- 交互说明：
  - 用户可基于指标创建阈值告警。
  - 支持趋势异常告警、数据刷新失败告警、数据延迟告警。
  - 用户配置告警接收人、通知渠道、触发频率和启停状态。
  - 系统记录告警历史。
- 异常与边界处理：
  - 告警计算依赖数据刷新结果，刷新失败时不触发指标阈值判断。
  - 通知发送失败时记录日志。
  - 高频告警需支持合并或静默策略，避免重复打扰。

### 2.17 审计与运维

- 功能目标：为管理员提供插件运行、资产使用和问题排查依据。
- 交互说明：
  - 系统记录操作日志，包括创建、编辑、删除、复制、分享、导出、转让。
  - 系统记录访问日志、查询日志、失败日志。
  - 管理员可查看资产使用统计，包括访问次数、最近访问时间、无人使用仪表盘。
  - 管理员可查看慢查询分析和插件运行状态。
- 异常与边界处理：
  - 日志保存周期和清理策略需支持配置。
  - 敏感字段不在日志中明文展示。

### 2.18 效能分析场景

- 功能目标：提供研发管理中常用的分析场景，作为仪表盘模板或推荐卡片来源。
- 交互说明：
  - 支持项目进度分析。
  - 支持迭代进度分析。
  - 支持完成工作项数量趋势。
  - 支持按月完成工作项累计趋势。
  - 支持需求关联 bug 数统计。
  - 支持需求关联不同类型工作项数量统计。
  - 支持缺陷创建量与解决量趋势。
  - 支持缺陷质量分析基础指标。
  - 支持工时类统计能力，前提是客户环境中工时数据可获取。
  - 支持团队成员维度分析，前提是成员、部门或用户组数据可获取。
  - 支持状态流转分析、状态停留时长分析、工作项交付周期分析、Reopen 次数分析、逾期工作项分析。
- 异常与边界处理：
  - 模板中的字段在客户环境不存在时，模板卡片进入待配置状态。
  - 历史状态类指标依赖变更记录，若不可获取，则不展示该指标或提示数据来源受限。

### 2.19 可用性与异常处理

- 功能目标：让用户在无数据、无权限、配置异常和查询失败时能够明确知道原因和下一步处理方式。
- 交互说明：
  - 支持无数据提示。
  - 支持权限不足提示。
  - 支持字段不存在提示。
  - 支持数据源不可用提示。
  - 支持查询超时提示。
  - 支持卡片配置异常提示。
  - 支持一键定位异常卡片配置。
  - 支持帮助说明或口径说明入口。
- 异常与边界处理：
  - 异常提示应区分用户可自行处理的问题和需管理员处理的问题。
  - 批量异常场景下，仪表盘顶部展示汇总提示，并允许定位到异常卡片。

### 2.20 可选 AI 增强

- 功能目标：在基础 BI 能力稳定后，增强自动洞察和分析效率。
- 交互说明：
  - 支持自动生成图表摘要。
  - 支持自动识别异常波动。
  - 支持推荐分析维度。
  - 支持推荐图表类型。
  - 支持自然语言问数。
  - 支持自动生成周报或月报分析摘要。
- 异常与边界处理：
  - AI 结果仅作为辅助分析，不替代原始数据。
  - AI 能力依赖模型服务、权限、安全合规和客户环境配置，需单独确认。

## 3. 验收口径

- AC1：插件启用后，具备 BI 仪表盘入口和插件配置页；插件停用后普通成员不可访问插件页面。
- AC2：用户可创建、编辑、删除、复制、归档、恢复仪表盘，并可在列表中按全部、我创建的、共享给我的、最近查看、收藏进行查看。
- AC3：仪表盘支持 owner、编辑者、查看者权限模型，owner 可转让仪表盘，转让后新 owner 可编辑。
- AC4：用户查看仪表盘时，统计结果不包含其在 ONES 中无权访问的数据。
- AC5：用户可创建数据集，并配置字段别名、字段类型、默认聚合方式、是否启用和口径说明。
- AC6：用户可配置指标、维度、筛选条件，支持多指标和常见聚合方式。
- AC7：用户可基于项目、迭代、工作项、状态、字段、成员、工时、关联关系等可获取数据创建卡片。
- AC8：仪表盘编辑态支持卡片新建、编辑、复制、删除、拖拽布局、尺寸调整和基础样式配置。
- AC9：数字卡支持小尺寸展示，不强制固定 3x3。
- AC10：系统支持文本卡、数字指标卡、柱状图、折线图、组合图、饼图、面积图、表格、透视表等基础图表。
- AC11：趋势图支持自然日展示和按月累计趋势。
- AC12：筛选能力支持时间范围、筛选条件组、模糊匹配、项目与迭代级联筛选、多维排序。
- AC13：全局筛选器可配置作用范围，支持作用于全部卡片或指定卡片。
- AC14：点击柱状图 bar、折线图点位、透视表单元格、数字卡可进入明细。
- AC15：明细表支持自定义表头、列宽拖拽、排序、筛选、字段显隐、工作项详情跳转。
- AC16：卡片展示加载中、无数据、失败、权限不足、配置异常等状态。
- AC17：仪表盘支持手动刷新、卡片级刷新状态、最近更新时间展示。
- AC18：系统按 100 张卡片仪表盘进行性能验证，并记录查询耗时和失败情况。
- AC19：支持仪表盘分享、订阅、导出能力；具体通知渠道和导出格式以可用性确认为准。
- AC20：系统记录关键操作日志和查询失败日志，管理员可按条件查询。
- AC21：效能分析模板或配置能力覆盖项目进度、迭代进度、完成趋势、缺陷趋势、需求关联 bug、工时统计、成员分析等场景。

## 4. 待确认项

### 4.1 业务问题

- 是否将 P0、P1、P2 全部纳入同一交付周期，还是按阶段拆分交付。
- 仪表盘入口最终放置在独立插件导航、效能管理导航、我的工作台，还是项目内入口。
- 首版是否必须支持仪表盘发布/草稿机制，还是可作为 P1 延后。
- 客户对“数据集”和“指标管理”是否需要独立页面，还是仅在卡片配置中内嵌。
- 多选字段拆分聚合是否按“命中选项分别计数”作为客户确认口径。
- 多指标趋势图的时间横轴口径是否统一采用自然时间。
- 导出格式首版需要支持 PDF、Excel、图片中的哪些格式。
- 告警、订阅、AI 洞察是否进入首版范围。
- 仪表盘模板需要预置哪些模板，以及模板字段口径是否由客户确认。
- 资产使用统计和审计日志的保存周期需要确认。

### 4.2 系统约束

- 本方案按 ONES Open Platform 1.0 插件设计，不使用 Open Platform 2.0 `.opkx`、`opkx.json`、Hosted API 或 Web SDK 能力。
- 插件能够分析的数据范围受 OpenAPI、ONESQL 或客户环境可用查询能力限制。
- 插件权限不扩大 ONES 原有数据权限，用户无权访问的数据不得在统计结果、明细或导出中出现。
- 工时、部门、用户组、工作项历史变更等数据是否可分析，取决于客户环境中数据是否存在且可获取。
- 通知、订阅、告警推送依赖客户环境可用通知渠道。
- 大批量导出、100 卡片仪表盘和复杂图表联动需进行专项性能验证。
- AI 增强能力依赖模型服务、安全合规、客户部署方式和数据授权，需单独确认。
